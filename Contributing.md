# MiMo One 贡献指南

> 欢迎来到 MiMo One！本项目为 ArtNh 的个人数字化资产库与桌面端实验平台，旨在将硬核的多智能体调度系统转化为直观、可控的现代桌面工作流。我们欢迎基于理性的技术探讨与 Pull Request 提交。

为了保持项目极高的代码质量、视觉美学与严苛的纪律性，请在贡献代码前务必仔细阅读以下指南。

---

## 🏗 本地开发环境设置

本项目基于 `Electron + Vite + React 18 + TypeScript + Tailwind CSS` 构建。

### 1. 环境准备
确保您的开发环境已安装：
- **Node.js** (推荐 v18 或以上)
- **Git**

### 2. 本地启动
```bash
# 克隆仓库
git clone https://github.com/ArtNh/MiMo-One.git
cd MiMo-One

# 安装依赖
npm install

# 启动桌面端调试环境
npm run dev:desktop
```
> **注意**：启动脚本会自动拉起 Vite 开发服务器，并在端口（默认 5174）就绪后自动启动 Electron 窗口。

---

## 🧩 核心 UI 模块开发指引

如果您计划对 UI 模块进行贡献，请遵循本项目的极简主义与理性空间美感设计（纯净 Light Mode）：

1. **记忆与进化控制台 (左侧)**
   - 负责可视化 Project Memory 与 Archival Memory。
   - 开发时需注意数据的流向与“The Dream State（睡眠进化）”呼吸动效的平滑度。
2. **多模式执行台 (中央)**
   - 核心的卡片折叠对话流。
   - **Max Mode (全托管模式)**：在开发超级输入舱时，请确保边缘泛蓝光反馈的性能表现，避免阻塞主线程。
3. **智能体监控抽屉 (右侧)**
   - 实时监控 Subagent 任务树与文件 Diff 热预览。
   - 请确保 Diff 视图的解析准确性与滚动性能。

## 🚨 核心开发纪律 (不可触碰的红线)

为了维护项目极其严苛的规范与时间线准确性，所有贡献者**必须无条件遵守**以下三大纪律。任何违反红线的行为都将导致 PR 被直接拒绝。

> **一、文档与文件纪律**
> 
> 所有在开发期间生成的开发手记、实施计划（Implementation Plan）、任务清单（Task）以及任何 AI 生成的分析与总结文档，**必须无条件存入项目根目录下的 `Markdown_Notes` 文件夹中**。
> 绝对严禁将这些文档散落、随意存放于项目根目录或其他源码目录中。

> **二、日志修改纪律**
> 
> 对于任何针对项目文档或日志文件的写入与维护，**必须强制采用“追加（Append）”模式**。
> 绝对禁止覆盖、删除或篡改历史记录内容。新增加的日志必须附带精确到秒的标准时间戳标记。

> **三、提交规范纪律**
> 
> 我们实行极其精密的定时版本控制标准。您的 Git 提交（Commit Message）必须**完全遵循**以下格式：
> `[Prefix(Object): Summary (YYYY-MM-DD HH:mm:ss)]`
> 
> *格式示例：*
> - `[feat(max-mode): 实现超级输入舱泛蓝光特效 (2026-06-11 20:30:15)]`
> - `[fix(archival): 修复记忆树折叠状态丢失问题 (2026-06-11 21:05:00)]`

## 🔄 提交 Pull Request 工作流

1. **Fork 本仓库** 到您的个人账户下。
2. 从 `main` 分支切出一个具备说明性的 `feature/*` 或 `fix/*` 独立开发分支。
3. 遵循上述核心纪律完成您的编码与文档记录。
4. 使用严格的带秒级时间戳格式进行 Git 提交。
5. 向我们的 `main` 分支发起 Pull Request，并在描述中提供最小可重现上下文与测试验证结果。

感谢您为本极客工作流注入新的智慧与代码，让我们在理性的技术探讨中共同进步。
