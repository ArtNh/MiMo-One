# 本地文件系统安全访问与隔离策略

---

### [2026-06-15 18:30:00] 安全控制规范

## 安全机制概述

为了保障用户主机文件的私密性与系统稳定性，在 Electron 桌面端及前端渲染层之间引入了多重隔离机制与严格的安全边界防御设计。

> 本规范定义了工作区扫描过程中的目录隔离、跨越防御、数据截断以及在非桌面端环境下的模拟退化策略。

---

## 核心安全控制策略

### 1. 物理目录白名单控制 (Directory Whitelist Control)
- **基准路径锚定**：扫描根目录严格限定为 Electron 运行的 `process.cwd()`（当前工作区根目录）。
- **递归黑名单过滤**：在进行深度优先（DFS）目录遍历时，对以下四类文件夹执行静默拦截，不予进入且不收集任何内容：
  - `node_modules`：第三方依赖库，避免产生数以万计的无关文件索引。
  - `.git`：版本控制元数据，防止泄露版本提交历史及内部配置文件。
  - `dist` 与 `dist-electron`：产品打包发布目录，防止对编译产物重复建立索引造成死循环。

---

### 2. 路径防跨越攻击校验 (Path Traversal Protection)
- **双重路径解析**：主进程通过 `path.resolve` 或 `path.normalize` 计算绝对路径。
- **前缀比对熔断**：在任何文件读取请求执行前，通过 `fullPath.startsWith(process.cwd())` 校验目标路径前缀。
  
> **熔断保护机制**：若目标物理路径计算结果位于项目根目录之外，主进程将立刻抛出 `Unauthorized file access path` 异常，强行熔断文件读取流，并拦截跨目录读取行为。

---

### 3. 数据截断与内存溢出防御 (Memory Exhaustion Prevention)
- **大小流式监测**：在获取文件详情时，先行获取文件整体大小，防止直接加载超大文件引起进程内存崩溃（OOM）。
- **1500字符安全摘要**：大模型仅需要提取关键的头文件、类型声明或代码逻辑结构作为 Context。因此主进程对读取的内容统一截取前 **1500** 字符作为摘要返回给渲染进程。

---

### 4. 浏览器环境无痛降级设计 (Graceful Fallback Design)
- **环境安全检测**：通过检查渲染层 `window.electron.ipcRenderer` 是否挂载，自动判断运行形态.
- **仿真模拟器回退**：在浏览器非 Electron 容器环境下，通过定时器与预置 Mock 数据构建一套无损的流式任务推进器。前端无任何报错，白屏风险为零。
