## 构建目标与流程概述

为了支持应用作为独立的跨平台二进制客户端进行分发，我们集成了 `electron-builder` 打包引擎，并制定了标准的一键式自动化打包管道。打包流程将前端静态资源编译（Vite + React）、桌面端主进程及预加载脚本编译、以及跨平台应用打包压缩（Electron Builder）进行流水线串联，保障发布产物的一致性。

---

## 核心配置说明 (electron-builder.json)

> [!IMPORTANT]
> **多资源随包依赖及路径保护**：
> - 大模型物理命令执行内核 `mimo-code` CLI 的可执行文件需要被放置在根目录的 `bin` 文件夹下。在打包时，打包引擎通过 `extraResources` 配置将其安全包含至包体内（即打包后的 `resources/bin`）。
> - 底层主进程在执行时将自动感知环境状态，对于 packaged 生产环境会直接重定向至 `${process.resourcesPath}/bin/mimo-code` 下执行。

## 跨平台打包构建指令

开发人员可以根据目标操作系统执行对应的打包命令。

### 1. 自动执行一键打包 (全平台)
通过执行以下整合脚本，将自动在根目录下创建生产优化构建并打包：
```bash
npm run build:desktop
```

### 2. Windows 平台构建命令
若需要在其他平台（或本地）仅针对 Windows 环境生成安装包（NSIS 可执行程序 .exe）：
```bash
npx electron-builder --win --x64
```
*构建产物：放置在 `dist-build/` 目录下的 `MiMo One Setup <version>.exe`。*

### 3. macOS 平台构建命令
在 macOS 环境下针对苹果操作系统生成 DMG 安装映像或 .app 绿色包：
```bash
npx electron-builder --mac --universal
```
*构建产物：放置在 `dist-build/` 目录下的 `MiMo One-<version>.dmg`。*

## 生产环境环境变量注入规范

在生产环境应用分发和运行期间，系统需要接入相关密钥和后端 API 端点。为了确保敏感数据的安全性与配置灵活性，规定了以下环境变量注入及读取链路：

| 环境变量名称 | 作用 | 生产环境注入形式 |
| :--- | :--- | :--- |
| **`OPENAI_API_KEY`** | OpenAI 官方接口访问鉴权密钥 | 在左下角「系统设置」面板中动态输入持久化，或在系统环境变量中声明 |
| **`API_BASE_URL`** | 大语言模型服务的反向代理或官方端点 | 在系统设置面板中持久化配置，默认使用 https://api.openai.com/v1 |
| **`MIMO_WORKSPACE_PATH`**| 内核在执行本地物理重构时的默认根工作区 | 由 Electron 在初始化时自动获取当前执行目录并绑定 |

> **物理隔离与优先级定义**
> 
> 在物理指令唤起（`run-mimo-command`）时，主进程会优先抓取 Zustand 配置 store 中的用户配置，并将这些配置作为 `extraEnv` 字段以进程环境变量的形式动态注入给拉起的子进程。这种设计保证了即使在完全打包的沙箱环境下，用户动态更改的 API 密钥与代理路径也能够秒级同步生效。
