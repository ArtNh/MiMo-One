`failed to create browser context: failed to resolve CDP URLs: get CDP version info: could not resolve IP for 127.0.0.1`

## 排查与诊断详情

### 1. 系统网络适配器与代理排查
通过在控制台执行网络环境诊断命令，我们发现了以下关键线索：
- **虚拟网卡接管**：本地网络适配器中激活了名为 `Meta Tunnel` 的虚拟网卡，其配置的 IPv4 地址为 `198.18.0.1`，DNS 解析服务器指向 `198.18.0.2`。
- **全局代理进程**：当前系统后台运行着名为 `èè.exe`（在 GBK 编码下显示，对应中文“猫猫.exe”，即 Clash 客户端）的全局代理代理服务，并开启了系统级的 TUN 虚拟网卡接管模式。
- **解析机制干扰**：由于 TUN 模式劫持了本地所有的网络和 DNS 查询，导致 IDE 底层所依赖的网络库在进行回环 IP（`127.0.0.1`）的反向或正向 DNS 解析时被错误地路由并拦截，从而引发了 `could not resolve IP for 127.0.0.1` 的底层网络异常。

我们使用原生 Node.js 代码对回环地址的解析进行了独立验证：

- 对 `localhost` 进行 Lookup 时，Node.js 默认将其解析为了 IPv6 的回环地址 `::1`。

### 3. Vite 服务状态与端口绑定验证
- **服务启动状态**：Vite 编译服务器在本地 `5173` 端口上已完全就绪，并且其默认监听绑定在 IPv6 回环地址 `[::1]:5173` 上。
