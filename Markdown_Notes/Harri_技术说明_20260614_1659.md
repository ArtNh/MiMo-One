# Harri 技术说明

> **概述**
> 本文件记录在 MiMo One 项目中首次集成的核心 AI 助手实体 **Harri** 的实现细节、设计思路与交互规范。所有描述均使用简体中文，遵循项目的理性版式美感规范。

---

## 1. 组件位置与结构

- **文件路径**：`src/components/Harri/HarriStateViewer.tsx`
- **导出对象**：`HarriStateViewer`（默认导出）
- **依赖**：React、Tailwind CSS

### 2. 运行状态枚举

```ts
export type HarriStatus = 'sleeping' | 'processing' | 'idle';

export const HarriStatusEnum = {
  Sleeping: 'sleeping' as HarriStatus,
  Processing: 'processing' as HarriStatus,
  Idle: 'idle' as HarriStatus,
};
```

- 初始默认状态为 **`sleeping`**（低功耗、休眠）
- 枚举提供统一引用，便于后续扩展与类型安全

### 3. 组件核心实现

- 使用 `useState` 管理内部状态，默认 `sleeping`
- `useEffect` 示例循环演示状态切换（实际业务请通过外部事件驱动）
- 根据不同状态渲染不同 UI：
  - **sleeping**：`Zzz...（Harri 正在休眠）`，配合 Tailwind `animate-pulse` 实现呼吸式占位动画
  - **processing**：蓝色文字 `Harri 正在处理任务`
  - **idle**：绿色文字 `Harri 空闲待命`

### 4. 样式方案（Tailwind CSS）

```html
<div className="text-gray-500 italic animate-pulse">Zzz...（Harri 正在休眠）</div>
<div className="text-blue-600">Harri 正在处理任务</div>
<div className="text-green-600">Harri 空闲待命</div>
```

- `animate-pulse` 为 Tailwind 提供的淡入淡出动画，已在项目中全局开启
- 通过 `text-*-600` 与 `italic` 等实现场景感知的视觉层次

### 5. 挂载位置

- 在 `src/App.tsx` 中的 **B 区（中央交互舱）** 顶部插入 `<HarriStateViewer />`
- 组件被包裹在 `section`，带有边框与背景，确保不干扰现有布局

### 6. 后续扩展建议

1. **外部状态控制**：提供 `status` Props 与 `onStatusChange` 回调，以便上层业务通过 WebSocket、IPC 或本地事件驱动状态变化。
2. **语音/文字交互**：结合 Electron 主进程的语音合成与识别模块，为 Harri 增加自然语言交互入口。
3. **低功耗优化**：在 `sleeping` 状态下可进一步降低渲染频率，仅保留最小的动画占位，减少 CPU/GPU 消耗。
4. **主题适配**：依据全局暗/亮色模式切换文本颜色，提升跨平台一致性。

### [2026-06-14 18:31:00] 全局状态注入与输入框数据流绑定升级

> **升级概要**
> 本次更新实现了 `HarriStateViewer` 与底部输入框之间的数据流联动。将 Harri 的运行状态从组件内部演示循环中剥离，提升至父组件 `App.tsx` 作为全局状态进行管理，使状态流转能够真实反映系统的后台计算与交互行为。

#### 1. 状态管理与组件通信

- **受控状态定义**：
  在 `src/components/Harri/HarriStateViewer.tsx` 中定义并导出了受控属性 `HarriStateViewerProps`：
  ```typescript
  export type HarriStatus = 'sleeping' | 'processing' | 'idle';

  interface HarriStateViewerProps {
    status?: HarriStatus;
  }
  ```
- **父子组件数据传递**：
  在父组件 `App.tsx` 中统一维护 `harriStatus`，并将其作为 Props 注入：
  ```tsx
  const [harriStatus, setHarriStatus] = useState<HarriStatus>('idle');
  
  // 渲染部分
  <HarriStateViewer status={harriStatus} />
  ```
- **向后兼容的内部演示循环**：
  组件内部依然保留 fallback 机制：若外部未传入 `status` 属性，则会自动降级启用定时演示循环，确保组件在隔离使用时的独立可用性。

#### 2. 输入事件拦截与流转逻辑

- **输入拦截机制**：
  在 `App.tsx` 的底部输入框 `input` 绑定键盘监听事件。当检测到按下 `Enter` 键且有有效文本输入时触发消息发送：
  ```typescript
  const handleKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
    if (e.key === 'Enter') {
      handleSend();
    }
  };
  ```
- **异步 Promise 状态流转链**：
  在 `handleSend` 触发消息发送后，系统将立即把状态设置为 `processing`。通过 Promise 链模拟后台接口的异步请求与耗时响应，并在 `finally` 回调中自动还原为 `idle` 状态：
  ```typescript
  const handleSend = () => {
    if (!inputValue.trim()) return;
    setHarriStatus('processing'); // 进入“搬砖中”状态
    setInputValue('');

    new Promise<void>((resolve) => {
      setTimeout(() => {
        resolve();
      }, 2000); // 模拟2秒的网络发送延迟
    }).finally(() => {
      setHarriStatus('idle'); // 发送结束后还原为“随时待命”状态
    });
  };
  ```
