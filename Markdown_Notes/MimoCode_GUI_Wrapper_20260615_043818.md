## 架构重构背景

由于架构方向的重大调整，MiMo One 客户端被彻底重构为 Mimo Code 原生 CLI 的 1:1 可视化桌面端外壳（GUI Wrapper）。去除了客户端本身的大模型适配逻辑，转而直接在后台通过进程控制管道来支配本地或系统的 `mimo` / `mimo-code` CLI，以此达成最纯正的原生交互体验。

---

## 物理进程管道与接轨设计

> [!IMPORTANT]
> **交互管道重映射机制**：
> - **输入重映射**：渲染进程 B 栏的文本框在按下回车发送后，内容直接通过 IPC 通道写入 `activeMimoProcess` 子进程的 `stdin`。
> - **输出重映射**：主进程通过流式监听 `stdout` 与 `stderr`，一方面将原始的、带有 ANSI 颜色控制码的字节码原汁原味地发送给 C 栏终端展示；另一方面，前端会将控制码进行正则清洗（去除 ANSI 逃逸控制符）并流式追加至 B 栏的哈里中枢应答气泡中。

## Mimo CLI 原生命令映射清单

外壳客户端直接将用户的界面点击行为转换为后台静默执行的 Mimo 原生命令：

| GUI 触发动作 | 映射底层 Native CLI 命令 | 作用及设计逻辑 |
| :--- | :--- | :--- |
| **保存系统设置** | `mimo config set <key> "<val>"` | 直接物理覆写 `~/.mimo_config`，并同步执行 config set 指令同步环境变量 |
| **新建项目** | `mimo init` | 物理初始化 Mimo 项目工作区，并在 C 监控中实时回显 stdout 编译步骤 |
| **新建会话** | `mimo session new` | 截获控制台吐出的 `sess_<hash>` 唯一会话 ID，并将其挂载至 Zustand 全局状态 |
| **智能对话流** | `mimo chat` (若有 Session，附加 `--session`) | 拉起常驻进程以实现 stdin / stdout 双向管道通信交互 |

---

## 双端兼容仿真降级保障

为了保障开发演示和纯前端运行的完整性，当应用在打包运行于非 Electron 浏览器环境、或者系统中未配置 `mimo` CLI 环境变量时：
1. **主进程**：捕获 `spawn` 异常并优雅地向渲染层上报 `fallbackSimulation` 指标。
2. **渲染层**：自适应切换到本地仿真内核。B 栏会重新起用内置的 LLM 模拟应答，C 栏会自动启动任务进度仿真器模拟日志吐出，确保客户演示在所有宿主系统下都能零阻碍完美展示。
